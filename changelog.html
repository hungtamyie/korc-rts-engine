kOrcEngine HTML5 RTS Engine Development Log

===20 June 2011===
---Project Start---

* Objective: develop an RTS game engine meant to run in browsers.
* This engine is built around the vision of the game inspiration in this design document: Admiral Zala
* As such, this engine is meant to allow flexibility of mechanics, and just wrap away all the quintessential RTS ui and gameplay elements
* The engine will be tailored to make the above game, while keeping in mind the needs of a generic RTS game. * If you want me to keep another flexibility in mind, contact me!

===03 July 2011===

* Defined basic game class: custom event handlers, game loop setup, and canvas constant setup
* Created unit class: capable of moving to target point, accelerating, decelerating, custom selection with selection-circles
* Listeners for mouse actions

===10 July 2011===

* Drag-select, shift-select units
* UI class that can draw map-grid
* Grid-based unit positioning as a set-up for pathing algorithms
* Map panning, including keyboard listeners
* Standard RTS controls implemented: removed context menu; right-click->move

===14 July 2011===

* Way-points (shift-queued movements)
* Path drawings: lines from current position to destination(s) (also as setup for pathing)
* Path-updating framework in place for unit avoidance using circumventing waypoints

===19 July 2011===

* Pathing adjusts its destination if another unit occupies it before it starts or before it reaches
* Group pathing for more than one unit makes nominal sense if the units arrive at sufficiently different times
Process:
Check if the opening waypoints are "pathed" waypoints, ie, from a previous path update
If true, then just check the final destination occupancy
If unoccupied, return
If occupied, find closest spot that is open
Else, replot path to the destination
Currently Known Bugs:
* If selected unit is right beside target location, and it is occupied, sometimes units stack
* If units are grouped, there is no squad management, and they will stack if all units arrive at almost the same time

===26 July 2011===

* Units can now attack each other
* When attacking, units launch bullets, which can be drawn in any custom fashion, but deal damage when they reach
* Attacked and destroyed units will be removed from the game entirely. There's no system for tracking these stats yet
* When viewed in a browser larger than the game space, the map now clips graphically and functionally

===28 July 2011===

* Added a Google button to both this changelog as well as the demo game!
* Fixed corner cases on pathing so that the units don't just completely stop in their tracks near the edges/corners *
